Getting Started with inBloom Tagger

  • version 1.2 release notes

    This section outlines the new features, bug fixes, and behaviors of Tagger version 1.2.

    Tagger complies with LRMI V1.1. Much of LRMI was adopted by the Schema.org/CreativeWork specification. Tagger is also compliant with Schema.org/CreativeWork, except for those LRMI properties that were not adopted by Schema.org.

  • version 1.1 release notes

    This section outlines the new features, bug fixes, and behaviors of Tagger version 1.1.

    Tagger complies with LRMI V1.1 specification.

  • welcome

    Welcome to inBloom Tagger Version 1.2 inBloom Tagger is a metadata-tagging tool that is designed to support both individual tagging and bulk processing of educational resources. This user guide provides detailed instructions on the use of this tool.

  • learn about inBloom Tagger

    The inBloom Tagger window is composed of a navigation bar at the top of the screen and three panes below the navigation bar. The user can use inBloom Tagger with limited functionality without signing in; however, for full functionality (including the ability to save the user’s work and publish the user’s learning resource metadata tags to the Learning Registry (LR)), the user must sign in with an inBloom username and password using the link at the top right of the screen.

  • create multiple resources using quick load

    The user can efficiently create multiple resources in a single action using the Quick Load feature. Once the resources are created, the user may edit them to add additional information.

Create Tags in Batches by Importing Multiple Resources

  • create an import file

    The user can bulk-upload educational resources into inBloom Tagger to create metadata tags that comply with the LRMI Specification.

  • import multiple resources from a CSV or JSON File

    Before import, please ensure that:

    1. The import file was constructed using the Import Data Rules
    2. The import file is saved as a .csv or .json format.

View, Export, and Publish Tags

  • view the output for a resource (or set of resources)

    The user may view the output for a resource (or set of resources) to verify that the information was captured and/or updated correctly.

  • export and save locally

    The user may locally save the output from an inBloom Tagger session in three formats

    • .csv file
    • .json file
    • .html file
  • publish to LR

    Resource metadata saved on the server is submitted to the database and indexed periodically.

    Note: Please use caution when saving files to the server. Records are identified by URL: if the user submits a record for a URL that already exists in the database, the new submission will overwrite the previous submission.
