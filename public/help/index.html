inBloom Tagger 1.1 User Guide

Getting Started with inBloom Tagger

  • version 1.1 release notes

    Agilix IdeaWorks is pleased to release this new version of Tagger, V1.1. It incorporates some new features and fixes a number of bugs related to importing of data. Please read through this document to understand these new features and behaviors of Tagger. Online help is also available.

    Tagger complies with the LRMI V1.0 Specification, which in turn extends the Schema.org/CreativeWork specification.

  • welcome

    Welcome to inBloom Tagger Version 1.2 inBloom Tagger is a metadata-tagging tool that is designed to support both individual tagging and bulk processing of educational resources. This user guide provides you with detailed instruction on the use of this tool.

  • learn about inBloom Tagger

    The inBloom Tagger window is composed of a navigation bar at the top of the screen and three panes below the navigation bar. You can use inBloom Tagger with limited functionality without signing in; however, for full functionality (including the ability to save your work and publish your resources to the Learning Registry (LR)), you must sign in with an inBloom username and password using the link at the top right of the screen.

Create Tags Manually

  • tag a new single resource

    Follow the instructions below to create a tag for your educational resource. The more detail that you provide about your resource, the more visible it will be to users searching for resources.

    As you are constructing your entries, keep in mind that inBloom Tagger accepts only well-formed data. Use the following chart to ensure that the data you enter is well-formed. For more information on the LRMI Specification, visit the LRMI Specification website.

  • create and tag multiple resources

    Many resources have multiple attributes that are common. By selecting multiple resources when you are creating the tags, all information that you enter will be applied to all selected resources. The following instructions provide the steps for one method of tagging multiple resources.

  • create multiple resources using quick load

    You can efficiently create multiple resources in a single action using the Quick Load feature. Once the resources are created, you may edit them to add additional information.

Create Tags in Batches by Importing Multiple Resources

  • create your import file

    You can bulk-upload educational resources into inBloom Tagger to create metadata tags that comply with the LRMI Specification.

  • import multiple resources from a CSV or JSON File

    Before you import, please ensure that:

    1. Your import file was constructed using the Import Data Rules
    2. Your import file is saved as a .csv or .json format.

View, Export, and Publish Tags

  • view the output for a resource (or set of resources)

    You may view the output for a resource (or set of resources) to verify that the information was captured and/or updated correctly.

  • export and save locally

    You may save output locally from an inBloom Tagger session in three formats

    • .csv file
    • .json file
    • .html file
  • publish to LR

    Files saved on the server are submitted to the database and indexed periodically.

    Note: Please use caution when saving files to the server. Records are identified by URL: if you submit a record for a URL that already exists in the database, the new submission will overwrite the previous submission.
