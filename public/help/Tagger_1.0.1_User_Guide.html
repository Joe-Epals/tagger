SLC TAGGER

Tagger 1.0.1 User Guide

  • Welcome

    Welcome to the Shared Learning Collaborative (SLC) Tagger Version 1.0.1. Tagger is a metadata-tagging tool that is designed to support both individual tagging and bulk processing of educational resources. This user guide provides you with detailed instruction on the use of this tool.

    Following is overview information to provide context of how Tagger fits into the Shared Learning Infrastructure.

  • Tagger Overview

    The Tagger Window is composed of a navigation bar at the top of the screen and three panes below the navigation bar. You can use Tagger with limited functionality without signing in; however, for full functionality (including the ability to save your work and publish your resources to the Learning Registry Index (LRI)), you must sign in with a Shared Learning Collaborative username and password using the link at the top right of the screen.

  • Tag a New Single Resource

    Follow the instructions below to create a tag for your educational resource. The more detail that you provide about your resource, the more visible it will be to users searching for resources.

  • Create and Tag Multiple Resources

    Many resources have multiple attributes that are common. By selecting multiple resources when you are creating the tags, all information that you enter will be applied to all selected resources. The following instructions provide the steps for one method of tagging multiple resources.

  • Import Multiple Resources from a CSV or JSON File

    You can easily upload multiple resources from a .csv or .json file. Create the import file using the instructions found here.

    Please note, you must save the completed upload file in either the .csv or .json format.

  • Create Multiple Resources using Quick Load

    You can efficiently create multiple resources in a single action using the Quick Load feature. Once the resources are created, you may edit them to add additional information.

  • View the Output for a Resource (or Set of Resources)

    You may view the output for a resource (or set of resources) to verify that the information was captured and/or updated correctly.

  • Export and Save Locally

    You may save output locally from a Tagger session in three formats

    • .csv file
    • .json file
    • .html file
  • Publish to LRI

    Files saved on the server are submitted to the database and indexed periodically.

    Note: use caution when saving files to the server. Records are are identified by URL: if you submit a record for a URL that already exists in the database, the new submission will overwrite the previous submission.
