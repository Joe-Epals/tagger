LRMI Tagger 1.0

LRMI Tagger V 1.0

  • Welcome

    Welcome to the Learning Resource Metadata Initiative (LRMI) Tagger Version 1.0. The LRMI Tagger is a metadata-tagging tool that is designed to support tagging of both individual and bulk processing of educational resources. This user guide provides you with detailed instruction on the use of this tool.

    Following is overview information to provide context to how Tagger fits into the Shared Learning Infrastructure.

  • Tagger Overview

    The Tagger Window is composed of three panes.

  • Tag a New Single Resource

    Follow the instructions below to create a tag for your educational resource. The more detail that you provide about your resource, the more visible it will be to users searching for resources.

  • Create and Tag Multiple Resources

    Many resources have multiple attributes that are common. By selecting mutliple resources when you are creating the tags, all information that you enter will be applied to all selected resources. The following instructions provide the steps for one method of tagging multiple resources.

  • Load Multiple Resources from a CSV File

    You can easily upload multiple resources from a .csv file. Download the .csv template from http://23.23.213.206/LRMI%20template.csv

    Please note, you must save the completed upload file in the .csv format.

  • Create Multiple Resources using Quick Load

    You can efficiently create multiple resources in a single action using the Quick Load feature. Once the resources are created, you may edit them to add additional information.

  • View the Output for a Resource (or Set of Resources)

    You may view the output for a resource (or set of resources) to verify that the information was captured and/or updated correctly.

  • Save the Output as an External File

    You may save output from a Tagger session in three ways:

    • Locally as a .csv file
    • Locally as a json file.
    • On the server as a json file

    Files saved on the server are submitted to the database and indexed periodically.

    Note: use caution when saving files to the server. Records are are identified by URL: if you submit a record for a URL that already exists in the database, the new submission will overwrite the previous submission.
